# 阶段 3: 骨架稿写作结构

这一份只服务阶段三。

## 结构选择规则

1. 用户明确指定结构时优先使用
2. 题材明显时按题材匹配
3. 题材不明显时从 2-3 个候选里选最贴切的
4. 连续发多篇时尽量别总用同一种开头

## 常用结构库

| `article_structure` | 适合题材 | 推进方式 |
|---|---|---|
| `data-first` | 榜单 / benchmark / 财报 / 融资 | 数字是什么 -> 为什么可信 -> 意味着什么 |
| `question-led` | 争议 / 真假难辨 / 用户困惑 | 提问题 -> 列证据 -> 给边界清晰的结论 |
| `contrarian` | 舆论一边倒但事实更复杂 | 先抛反直觉判断 -> 再给证据 |
| `teardown` | 新框架 / 新协议 / Agent 架构 / SDK | 总览 -> 分层拆解 -> 设计取舍 |
| `field-notes` | 教程 / 踩坑 / 实测体验 | 我试了什么 -> 哪里顺 -> 哪里坑 |
| `timeline-news` | 热点新闻 / 公司动作 / 安全事故 | 时间线 -> 角色 -> 影响 |
| `playbook` | 方法论 / 技巧合集 / 入门指南 | 适合谁 -> 最短路径 -> 关键动作 |
| `case-file` | 翻车 / 处罚 / 商业事故 | 事实 -> 证据链 -> 问题出在哪 |

## 开头钩子库

| `opening_hook` | 说明 |
|---|---|
| `hard-number` | 用具体数字、金额、比例、版本号开头 |
| `sharp-question` | 用读者真会问的问题开头 |
| `contradiction` | 并列两个冲突事实 |
| `quote-first` | 用当事人原话、issue、commit message 开头 |
| `artifact-first` | 用日志、命令输出、榜单条目开头 |
| `timeline-first` | 用具体日期或小时级时间线开头 |
| `scene-first` | 用一个短场景开头，不要铺太长 |

## 默认骨架

```markdown
# 标题

> 摘要引言

## 开篇

![配图描述](placeholder)

## 小节一

## 小节二

## 小节三

## 写在最后
```

## 行内标色约定

| 标记 | 用途 |
|---|---|
| `**文本**` | 主强调 |
| `==文本==` | 黄色高亮 |
| `++文本++` | 蓝色高亮 |
| `%%文本%%` | 粉色高亮 |
| `&&文本&&` | 绿色高亮 |
| `!!文本!!` | 红色强调 |
| `@@文本@@` | 蓝色强调 |
| `^^文本^^` | 橙色强调 |

不要整篇只用一种 `**加粗**`。
